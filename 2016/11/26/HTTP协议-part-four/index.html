    4.返回结果的HTTP状态码

    状态码有这些类别：

    -----------------------------
1XX  Infomational 信息正在处理
-----------------------------
2XX  Success  请求正常处理完成
-----------------------------
3XX  Redirection  需要附加操作
-----------------------------
4XX  Client Error 无法处理请求
-----------------------------
5XX  Server Error 处理请求出错
-----------------------------

    HTTP状态码种类繁多(60+)，但实际经常使用到的只有14种.接下来我们来介绍这些。

    2XX 成功

    表示请求被正常处理。

    200 OK

    最常见到的状态码，表示请求被正常处理，根据请求方法返回对应的内容。

    206 Partial Content

    客户端只进行了范围请求。

    204 No Content

    请求被正常处理，但返回不带实体的主体，并不允许返回任何实体的主体，一般在只需要客户端向服务器发送消息，服务器不需要对客户端发送的时候使用。

    3XX 重定向

    301 Moved Permanently

    请求的资源被永久分配到了新的URI，以后应使用新的URI访问，如果在最后忘记添加斜杠：http://www.example.com/sample，就会产生301状态码。
    标准禁止将POST方法改变成GET方法，但是没人理他，几乎所有浏览器都会改成GET。

    302 Found

    请求的资源被临时分配到了新的URI，本次应使用新的URI访问。
    标准禁止将POST方法改变成GET方法，但是没人理他，几乎所有浏览器都会改成GET。

    303 See Other

    请求的资源存在另一URI，应使用GET方法定向资源。303与302有相同的功能，但他明确表示使用GET。

    304 Not Modified

    当客户端发送附带条件的请求，服务器允许访问请求但是未满足条件时，返回此状态码。

    307 Temporary Redirect

    与302有相同的含义，307会遵照标准，不从POST变成GET，但对于响应处理，不同浏览器有不同的情况。

    4XX 客户端错误

    400 Bad Request

    表示请求报文中存在语法错误。浏览器对待此状态码与200 OK一样。

    401 Unauthorized

    第一次返回401，响应中包含WWW-Authenticate首部用以质询(Challenge)用户信息，浏览器会弹出认证窗口。

    第二次返回401，说明用户认证失败。

    403 Forbidden

    表明对资源的请求被服务器拒绝，服务器端没有必要给出拒绝的理由，如果要的话，可以在实体的主体部分对原因做出描述。

    造成403的原因主要有，未获得授权，访问权限出现问题等等。

    404 Not Found

    服务器无法找到请求的资源，也可用在服务器拒绝请求并不想说明理由的时候。

    5XX 服务器错误

    500 Internal Server Error

    服务器端在执行请求时发生错误，也有可能是Web应用存在Bug或临时故障。

    503 Service Unavailable

    服务器处于超负载状态或正在停机维护，无法处理请求。

    不少状态码响应是错误的！
    即是说：

    状态码和状况的不一定一致！

    5.与HTTP协作的Web服务器

    虚拟主机

    一台HTTP服务器中可以托管多个域名，这是利用了虚拟主机的功能。虽然物理上只有一台服务器，但却可以让多个域名运行各自不同的网站。

    在客户端向DNS发送了域名后，DNS服务会将他解析成IP地址，即是说，请求发送到服务器时，已经是IP地址的形式。若多个域名部署在同一台服务器上，他们就会有相同的IP地址。所以为了搞清楚客户访问的到底是哪一个，请求报文中Host首部会完整指定主机名和域名URI。

    代理、网关、隧道

    在通信线路中有这些个用于通信数据转发的应用程序和服务器：代理、网关、隧道。

    代理(Proxy)

    代理可以按照基准分为这样两类：
    1.是否使用缓存
    2.是否修改报文

    在细分这两类之前，还是先看一下代理的基本使用。

    代理服务器的基本行为就是接受客户端发送的请求并将之转发给其他的服务器(当然也包括代理)。代理服务器不会改变请求的URI。

    HTTP通信过程中，每经过一个代理服务器，都会追加Via首部字段来标记经过的主机信息。

    缓存代理

    代理转发响应时，缓存代理会从服务器上将资源副本拷贝到代理服务器上，当代理再次收到相同资源的请求的时候，就可以不经源服务器，而将资源副本作为响应返回。

    缓存是存在有效期限的，因此代理也会因客户端的要求，向源服务器确认是否资源更新。如更新，会用新的资源覆盖旧的。

    这样便可以减少网络带宽的流量。

    透明代理

    转发请求或响应时，对报文进行加工的代理叫做非透明代理。不对报文进行任何加工的叫做透明代理(Transparent Proxy)。

    设置组织内部的代理服务器可以做到针对特定URI访问的控制。

    网关(Gateway)

    网关和代理的工作机制十分相似，但网关可以是通信线路上的服务器使用非HTTP协议服务。如此便可以提高通信的安全性。

    因为网关就像拥有资源的源服务器一样对请求做处理。所以很多时候，客户端自己都不会察觉到，自己的通信目标是一个网关。

    隧道(Tunnel)

    隧道本身不会去解析HTTP请求，会将受到的保持原样中传给服务器。隧道建立起来安全的通信线路，使得相隔甚远的双方进行安全通信。

    隧道会在连接断掉时自动结束。
