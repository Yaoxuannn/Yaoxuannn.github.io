1.this漂移

在使用JS进行OOP设计的时候，一定要注意this的漂移问题，解决方法之一就是在声明函数的函数体开头加上

var self = this;

这样的语句。

2.setTimeout的一种妙用

使用定时器（Timeout）来解决事件多次触发的问题，通俗的说就是使得事件在满足触发条件且状况稳定的情况下才发生。

3.事件冒泡

个人认为，在任何时候都应该优先设置阻止事件冒泡

e.stoppPropagation();//阻止事件冒泡

另外顺便，提一下e.preventDefault();也是一个比较优先考虑的设置。

4.清空数组

当需要清空数组的时候，便利的方法有：

array.length = 1.
array = [];

5.调整图片

图片大小大于可视区域的时候，设置缩放比来控制大小正常显示。（一个觉得挺好的算法：）

var scale = Math.min(winWidth/picWidth, winHeight/picHeight, 1);

6.jQuery中的break

在jQuery中嵌套循环中使用return false;来跳出循环，相当于break;

7.移动动画

控制随机方向可以使用：

(Math.floor(Math.random() * 5 + 5)) * Math.pow(-1, Math.floor(Math.random() * 100))
 前半部分控制速度                    | 后半部分控制方向：数值越大，随机性越大

8.Range的坑

HTML5中新增的表单控件range在为其添加事件时，要注意change事件只有在改变确定下来的时候才会触发，所以为了获得即时效果要添加mousemove事件。

9.快速数组降维

通过[].concat.apply([], 2DArray to be flatten)来对二位数组降维。

如果是多维数组，可以使用这样的方法：

function flatten(arr) {
  var res = [];
  arr.forEach(function (val,idx) {
    if(Array.isArray(val)) {
      val.forEach(argument.callee)
    }
    else {
      res.push(val);
    }
  })
  return res;
}

10.DOMRect

element.getBoundingClientRect()方法，可以返回一个DOMRect对象，这个对象中包含了调用元素的大小和它相对于当前视口(viewport)的位置。

需要注意的是，由于参考物是当前的视口，所以当窗口滚动时，部分属性会变动。

所以，在使用时使用Element.getBoundingClientRect().top + window.scrollY来获取绝对top值。

DOMRect中的属性有(所有的属性都是float型且只读)：

bottom Y-coordinate, relative to the viewport origin, of the bottom of the rectangle box.

height Height of the rectangle box (This is identical to bottom minus top).

left X-coordinate, relative to the viewport origin, of the left of the rectangle box.

right X-coordinate, relative to the viewport origin, of the right of the rectangle box.

top Y-coordinate, relative to the viewport origin, of the top of the rectangle box.

width Width of the rectangle box (This is identical to right minus left).

x X-coordinate, relative to the viewport origin, of the left of the rectangle box.

y Y-coordinate, relative to the viewport origin, of the top of the rectangle box.

属性之间的关系有：

height = bottom - top;
width = right - left;

11.JS的动画最终解决方案

JS动画的最终解决方案是requestAnimationFrame()。

这个方法接受一个参数callback,返回一个的ID值，通过传入这个ID值可以来实现动画的停止等等.

requestAnimationFrame()的原理和setInterval()差不多，都是递归的调用，不同的地方是，这个方法会告诉浏览器，执行一系列动画，
并且执行的时间间隔是动态的，不同于setInterval()，这一解决方案会将函数体的执行时间也动态的考虑进去，最快的频率是60
