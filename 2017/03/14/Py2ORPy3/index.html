Python2 OR Python3?

ATTENTION: this latest update time of this original article is Jan. 29, 2015.
All translate by : Justin13
Translate from: http://learntocodewith.me/programming/python/python-2-vs-python-3/
Please contact to the author if you want to use it.

1.总体概要

在python的wiki上已有很详尽的关于Py 2.7和Py 3.3(目前最新为3.7 for dev)的区别，各有优劣。使用哪一个完全取决于你用来干什么，总结wiki的一句话：“Python 2.X是遗留产物。现在和将来应使用Python 3.X – wiki”

2.关于打印语句

在全世界最大的同性交友平台神社区Stack Overflow上，有很多关于此的讨论。

最显著的区别可能就是print语句的书写方式。同样的脚本在不同的版本上是不能运行的，你只能两者选一。

即：print 不再是一个语言结构.
Print Is A Function
在3.X中,你需要使用print("...")，而不是print "..."
(然而本人在2.7的解释器上两种都可以跑-_-#)

3.最重要的一点：Python 2 拥有库

在print语句的区别之上，网上很多人都提及了Python 2.7的库支持。

一篇在Medium上的文章，标题为 “Python 3 正在毁灭 Python这门语言”，很显然，作者 Stephen A. Goss, 强烈反对迁移至 Python 3 并且保持使用 Python 2，他同样指出第三方库是Python 2最强大的力量之一。

4.为什么一些人认为 Python 3.X 更好

在这个 talk 中，Brett Cannon，供职于Python，认为Python 3.3是Python语言的全部.
他承认自己是有明显的偏见，但仍然在这个一小时四十分钟的演说中阐述了观点。

除去这位供职于Python的哥们的观点，在网上亦有许许多多的相类似的观点。其中有这么一个我在 Quora 上看到的

3.X版本的最大优点就是它最先进，走在技术的前端。
所以所有的新特性都将会在此版本中实现，而不是在添加到2.X中。所以，几年下来，这将会成为开发者使用的最成熟的分支。

基本上，Python 3将是大势所趋，但Python 2仍然拥有远比3.X多的文档。

5.作为初学者，应该选择哪一个

如果你是像我一样的初学者，像我，你会在想:”嗯。可以。但我应该选择哪一个呢？”因为当你还是编程萌新的时候，你还在试图理清脑中关于函数的东西，而不要去对同一种语言不同版本之间的微妙之处进行深入分析。

在Stack OverFlow的一个早期讨论上提及到：

Python 3是一个优美，更加如一的语言，但是，仅仅有非常有限的第三方库支持，似乎在几年后仍会如此。所以，所有的主流框架仍然会在Python 2上运行，并且将会持续一段很长的时间。因此，如果你现在开始学习Python，你应当学习Python 2，因为这将会是你最终使用的。

同时，你也应该开始学习Python 3。因为不管怎么说，这是这门语言的未来发展。

但是，我仍然坚持以2.7开始是更加明智的，因为大量的文档是你尽快开上车。

另外，目前的系统(最新的Cent OS 7中绑定的Python版本为2.7)有很多内置2.X版本。

6.总结：Python 2 是赢家…目前如此

!!在作者的最新更新中，已经建议学习并使用Python 3!!

现在我将继续学习2.7。 尽管看来2.x和3.x之间的差异不大.我的主要目标是只学一种编程语言,而不想陷入在两个版本中做选择的麻烦中。

不管怎么说，他们看起来都可以接受。

在2015年1月29日的更新：

StackOverflow上的讨论已经将观点移向了开发者应开始使用Python 3

如果我现在开始学Python，我将会从Python 3开始。

题外话

选择哪一种版本的最终决定，并不是很重要，重要的是Research in yourself.

这篇翻译仅仅是为了提供一个参考方向，这之中我认为最重要就是第三方库的依赖了，当然到了现在，Python3的支持已经非常好了，并且3.X版本在数据科学，大数据，机器学习等科技前沿，增加了很多支持和独有的特性。

再多说一句：推荐使用Python 3😂😂
