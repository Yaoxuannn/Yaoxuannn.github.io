学习使用pymysql来进行数据库的操作

pymysql是遵循Python的DB-API 2.0开发的库,用来及逆行数据库的操作.

简单说一下Python的DB-API的标准,

属性 描述
apilevel 需要适配器兼容的DB-API版本
threadsafety 本模块线程安全级别
paramstyle 本模块sql语句参数风格
connect() Connect()函数
(Errors) 各种Error

这其中的数据类型有以下:

  • apilevel 该值默认为1.0, pymysql是2.0
  • threadsafety 这是一个整型值, 代表以下含义(pymysql是1):
    • 0 不支持线程安全
    • 1 最小化支持, 仅仅可以共享模块
    • 2 适度支持, 可以共享模块和连接
    • 3 完整的线程安全支持, 模块,连接,游标都可以共享
  • 所谓paramstyle顾名思义,就是指占位符的语法风格
  • connect函数,会在下面进行说明.

另外,DB-API提供的异常类标准如下:

  • Warning 警告异常基类
  • Error 错误异常基类
    • InterfaceError 数据库接口错误
    • DatabaseError 数据库错误
    • DataError 理数据时出错
    • OperationalError 数据库执行命令时出错
    • IntegrityError 数据完整性错误
    • InternalError 数据库内部出错
    • ProgrammingError SQL 执行失败
    • NotSupportedError 试图执行数据库不支持的特性

如果曾经使用过PHP的数据库抽象层(PDO),那么你就会发现python操作数据库是如此的简单,因为他们的API简直一模一样 :)

再有了一些了解后,还是先从最基础的来说一说吧.

原理

在python中数据库是通过适配器的方式进行访问的，适配器是一个python模块，使用他可以与关系数据库的客户端库（通常是使用C语言编写的）接口相连.

通过调用适配器提供的接口, 指令被发送到数据库, 数据库进行响应, 接着发送结果集(如果出错则发送错误码和错误消息), 适配器受到响应,返回给上层.

Connection

应用如果想要操作数据库, 就要先建立连接, 没有连接, 一切都无从说起, 当一个连接建立,我们就会得到一个Connection对象,这个对象会负责创建并返回一个游标(下面会说)

与文件读写等等这些和上下文有关的对象一样, 在一个connection使用结束后应该及时关闭(一定要先关闭cursor)

另外,Connection还要对支持事务的数据库提供提交和回滚的支持再加上错误处理, 这些就是标准规定的一个Connection对象应该包含的方法.

Cursor

游标对象可以说是应用与数据库交流的主力, 就是他讲请求发送到数据库, 再从数据库接受回应.

游标的属性和方法太多了,在这里仅列举较为核心的:

最核心和重要的方法就是execute()和fetch*()方法. 一个用来进行数据库查询,一个用来获取查询结果.

其中fetch一家人有:

  • fetchone()
  • fetchall()
  • fetchmany([size=cursor.arraysize])
    • 这里出现了一个arraysize属性用来充当默认many查询的条数, 默认为1

在每次进行过查询后,都会返回受影响的条数,这个数值也可以通过调用cursor.rowcount

在得到结果集后, 也可以使用__iter__得到cursor的迭代器,使用这个迭代器来进行遍历, 一次next()的操作其实就相当于是进行了一次fetchone()操作.

另外, 使用cursor.connection得到连接对象, 使用cursor.description得到当前游标的活动状态.

使用完游标要及时关闭, 游标和连接都是不可共享的.
