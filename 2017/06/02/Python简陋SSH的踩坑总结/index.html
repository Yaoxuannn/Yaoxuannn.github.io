Ques 3: 同时解决标准输出和标准错误输出

原先初版使用的命令执行是通过os模块的popen方法执行的.

os.popen(cmd, mode='r', buffering=-1)

一般使用的话, 只需要向popen传送一个参数cmd就行了. 这样就会返回指令处理的结果对象(<class 'os._wrap_close'>), 接着直接进行读(read, readline, 什么的)操作就行了, 这会自动调用这个对象的_stream属性, 也就是和你打开文件时获得的文件对象的同样的类(<class '_io.TextIOWrapper'>).

但是,在实际进行测试的时候发现了问题,popen会将结果输出到对象中, 但是如果命令发生错误, popen会直接将stderr输出, 也就是屏幕., 如果非要捕获的话, 就要在命令后面加上各种>>管道, 看起来很不舒服, 还要再进行管道流的提取, 很麻烦.

所以.在这种情况下, 就要选择subprocess模块下的Popen
方法了.然而, 在查阅文档的时候发现, os.popen的实现就是依靠subprocess.Popen, 当然底层还是C的函数popen.

之所以选择subprocess.Popen,就是因为他能方便的进行重定向.subprocess.Popen的方法签名是:

subprocess.Popen(args, bufsize=-1, executable=None, stdin=None, stdout=None, stderr=None, preexec_fn=None, close_fds=True, shell=False, cwd=None, env=None, universal_newlines=False, startupinfo=None, creationflags=0, restore_signals=True, start_new_session=False, pass_fds=(), *, encoding=None, errors=None)

( 善意的微笑🙂.

subprocess.Popen能用的到的方法签名是:

subprocess.Popen(args, stdin=None, stdout=None, stderr=None, shell=False)

Ques 4: 缓冲区大小限制

一般在进行Socket的收发时, 缓冲的大小设定都是1024, Python官方的建议是不要超过8192. 但事实上, 这个缓冲的大小最大是可以达到10M的.

初版的SSH完成后发现这样的问题: ifconfig的显示结果不全, 接着在执行下一个命令的时候, 显示的又是上一次没有显示完的信息.

全乱套了!🙁
